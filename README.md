# Piñata

[![Maintenance](https://img.shields.io/badge/Maintained%3F-yes-green.svg)](https://github.com/SeanNachapat/Pinata/graphs/commit-activity) [![License: MIT](https://img.shields.io/badge/License-MIT-green.svg)](https://opensource.org/licenses/MIT)

We are an open-source virtual IoT device service and Python package designed for evaluating anomaly detection cores and building IoT pipelines. Piñata allows working with fake sensor devices that behave exactly like real hardware, providing realistic data streams and an **event injection API** to trigger failures on demand.

Piñata acts as the perfect companion data source for ML pipelines and anomaly detection libraries.

Piñata consists of the following core features:

1. **Device Presets:** Instantly spin up complex, physically realistic virtual devices (starting with an Industrial Motor).
2. **MQTT Integration:** Piñata behaves exactly like real hardware by acting as an MQTT client. It also includes a zero-setup test broker for rapid prototyping.
3. **REST API & WebSockets:** Tap into the live data stream directly or inject anomalies on the fly.
4. **Ground Truth Labels:** Every data point is tagged with a label, making it trivial to train and validate anomaly detection models.

## Device Presets

Piñata comes with real-world device presets. The first available preset is the **Industrial Motor**. The datasets generated by these presets represent a multivariate time series. Columns (sensors) in the Industrial Motor stream include:

- `datetime` - Represents dates and times of the moment when the value is generated (YYYY-MM-DD hh:mm:ss.ms)
- `RPM` - Shows the revolutions per minute of the motor
- `VibrationX`, `VibrationY`, `VibrationZ` - Shows the vibration amplitude across 3 axes
- `Temperature` - Shows the temperature of the motor body (Celsius)
- `Current` - Shows the amperage drawn by the electric motor (Ampere)
- `anomaly` - Shows if the current point is anomalous (0 or 1) based on injected failures
- `anomaly_type` - String identifier of the active failure (e.g., `bearing_wear`, `mechanical_jam`) or `None`

## Installation

1. Install Python 3.10+
2. Install [poetry](https://python-poetry.org/docs/) package manager:
   ```bash
   curl -sSL https://install.python-poetry.org | python3 -
   ```
3. Clone the repository and install Piñata dependencies:
   ```bash
   git clone https://github.com/SeanNachapat/Pinata.git
   cd Pinata
   poetry install
   ```

## Quickstart Notebooks & Examples

The easiest way to get started is to use the provided template script in `examples/basic_template.py`.

```bash
python examples/basic_template.py
```

This script contains the following core boilerplate to spin up a virtual Industrial Motor and start the server:

```python
import asyncio
from pinata.devices import IndustrialMotor
from pinata import Server

async def main():
    # Spin up an Industrial Motor with a temporary zero-setup MQTT broker
    device = IndustrialMotor(start_test_broker=True)
    
    # Expose the REST/WebSocket API and start streaming
    server = Server(device=device, port=8000)
    await server.start()

if __name__ == "__main__":
    asyncio.run(main())
```

Once running, you can inject an anomaly via REST:
```bash
curl -X POST http://localhost:8000/api/inject \
    -H "Content-Type: application/json" \
    -d '{"anomaly_type": "bearing_wear", "duration_seconds": 60}'
```

## Web Landing Page & Interactive Playground

We have built a gorgeous, interactive Next.js marketing landing page and hardware telemetry playground inside the `web/` directory. This web app displays features, provides copy-paste quickstarts, and hosts an interactive simulator running physically realistic devices (Motor, Turbine, HVAC) and fail states directly inside the browser window.

### Running Locally
To launch the web app on your local machine:
1. Navigate into the web folder:
   ```bash
   cd web
   ```
2. Install npm packages:
   ```bash
   npm install
   ```
3. Start the development server:
   ```bash
   npm run dev
   ```
4. Open [http://localhost:3000](http://localhost:3000) in your web browser.

### Deploying to Vercel
This landing page is pre-configured and optimized to be hosted on **Vercel** for free:
1. Push your repository to **GitHub**.
2. Go to the [Vercel Dashboard](https://vercel.com/) and import your project.
3. In the project settings, set the **Root Directory** to `web`.
4. Click **Deploy**. Vercel will automatically build the Next.js bundle and host the interactive landing page.
